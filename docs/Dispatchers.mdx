Dispatchers synchronize incremental policy changes using consistency algorithms like Raft, ensuring consistency across all Enforcer instances. They enable easy creation of distributed Casbin clusters.

Dispatcher functionality consists of two parts: Casbin-integrated methods called internally by Casbin, and dispatcher-specific methods for initialization, dynamic membership, and configuration changes.

:::note

Dispatchers ensure consistency of Casbin Enforcers during runtime operations. All instances must have consistent initial policy states; inconsistent initialization will cause unexpected behavior.

:::
