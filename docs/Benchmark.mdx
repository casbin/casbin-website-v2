Casbin policy enforcement overhead has been benchmarked in [model_b_test.go](https://github.com/casbin/casbin/blob/master/model_b_test.go). The testbed configuration is as follows:

```text
Intel(R) Core(TM) i7-6700HQ CPU @ 2.60GHz, 2601 Mhz, 4 Core(s), 8 Logical Processor(s)
```

Benchmark results were obtained by executing `go test -bench=. -benchmem` (op = one `Enforce()` invocation, ms = millisecond, KB = kilobytes):

| Test case                 | Rule size                                | Time Overhead (ms/op) | Memory Overhead (KB/op) |
|---------------------------|------------------------------------------|-----------------------|-------------------------|
| ACL                       | 2 rules (2 users, 0 roles)                        | 0.016                 | 5.65                    |
| RBAC                      | 5 rules (2 users, 1 role, 0 domains)                | 0.022                 | 7.52                    |
| RBAC (small)              | 1100 rules (1000 users, 100 roles)       | 0.164                 | 80.62                   |
| RBAC (medium)             | 11000 rules (10000 users, 1000 roles)    | 2.258                 | 765.15                  |
| RBAC (large)              | 110000 rules (100000 users, 10000 roles) | 23.917                | 7606.00                 |
| RBAC with resource roles  | 6 rules (2 users, 2 roles)               | 0.021                 | 7.91                    |
| RBAC with domains/tenants | 6 rules (2 users, 1 role, 2 domains)     | 0.033                 | 10.76                   |
| ABAC                      | 0 rule (0 user)                          | 0.008                 | 2.33                    |
| RESTful                   | 5 rules (3 users)                        | 0.045                 | 91.77                   |
| Deny-override             | 6 rules (2 users, 1 role)                | 0.023                 | 8.37                    |
| Priority                  | 9 rules (2 users, 2 roles)               | 0.016                 | 5.31                    |
