Casbin provides two main solutions for data permission filtering: using implicit assignment APIs or the `BatchEnforce()` API.

### 1. Query Implicit Roles or Permissions

When a user inherits roles or permissions through an RBAC hierarchy rather than direct assignment in policy rules, this is called "implicit" assignment. To query these implicit relationships, use the following APIs: `GetImplicitRolesForUser()` and `GetImplicitPermissionsForUser()`, instead of `GetRolesForUser()` and `GetPermissionsForUser()`. For more details, see [this GitHub issue](https://github.com/casbin/casbin/issues/137).

### 2. Use `BatchEnforce()`

`BatchEnforce()` evaluates multiple requests simultaneously and returns results as a boolean array.
