Менеджер ролей используется для управления иерархией ролей RBAC (картирование ролей пользователя) в Касбине. Менеджер ролей может получить данные ролей из правил политики Casbin или внешних источников, таких как LDAP, Okta, Auth0, Azure AD, и т.д. Мы поддерживаем различные реализации ролевого менеджера. Чтобы сохранить легкий вес, мы не помещаем код ролевого менеджера в основную библиотеку (кроме менеджера по умолчанию). Полный список ролевых менеджеров Касбина приводится ниже. Любой вклад сторонних разработчиков в новый менеджер приветствуется, пожалуйста, сообщите нам и я поместим его в этот список:)

<Tabs groupId="langs">

| [Менеджер ролей по умолчанию (встроен)](https://github.com/casbin/casbin/blob/master/rbac/default-role-manager/role_manager.go) | Casbin                                           | Поддержка иерархии ролей, хранящейся в политике Касбина                                                                                                        |
| [Менеджер ролей сессии](https://github.com/casbin/session-role-manager)                                                         | [EDOMO Система](https://github.com/edomosystems) | Поддерживает иерархию ролей, хранящуюся в политике Касбин, с хронологическими сессиями                                                                         |
| [Okta ролей менеджер](https://github.com/casbin/okta-role-manager)                                                              | Casbin                                           | Поддерживает иерархию ролей хранится в [Okta](https://www.okta.com/)                                                                                           |
| [Авторский менеджер ролей](https://github.com/casbin/auth0-role-manager)                                                        | Casbin                                           | Поддерживает иерархию ролей, сохраненную в [Auth0](https://auth0.com/) [Authorization Extension](https://auth0.com/docs/extensions/authorization-extension/v2) |
