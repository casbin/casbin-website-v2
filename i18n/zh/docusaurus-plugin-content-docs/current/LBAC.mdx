## 什么是 LBAC？

**LBAC（基于标签的访问控制）** 是一个形式化模型，可以在一个框架内同时实施**机密性**和**完整性**。本页展示了一种 Casbin 实现方式：主体（Subject）和对象（Object）各自拥有机密性级别和完整性级别；匹配器（Matcher）在这两个维度上实施读/写规则。

## 模型

```conf
[request_definition]
r = sub, subject_confidentiality, subject_integrity, obj, object_confidentiality, object_integrity, act

[policy_definition]
p = sub, obj, act

[role_definition]
g = _, _

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = (r.act == "read" && r.subject_confidentiality >= r.object_confidentiality && r.subject_integrity >= r.object_integrity) || (r.act == "write" && r.subject_confidentiality <= r.object_confidentiality && r.subject_integrity <= r.object_integrity)
```

当主体的机密性和完整性均 ≥ 对象时允许读取；均 ≤ 对象时允许写入。这样，机密性遵循类似 BLP 的规则，完整性遵循类似 Biba 的规则，均在同一个匹配器中实现。

## 请求示例

以下示例展示了此 LBAC 实现如何评估请求：

```text
# 正常读取操作（允许）
admin, 5, 5, file_topsecret, 3, 3, read      # admin（机密:5，完整:5）读取 file_topsecret（机密:3，完整:3）- 允许
manager, 4, 4, file_secret, 4, 2, read       # manager（机密:4，完整:4）读取 file_secret（机密:4，完整:2）- 允许
staff, 3, 3, file_internal, 2, 3, read       # staff（机密:3，完整:3）读取 file_internal（机密:2，完整:3）- 允许
guest, 2, 2, file_public, 2, 2, read         # guest（机密:2，完整:2）读取 file_public（机密:2，完整:2）- 允许

# 读取操作违规（拒绝）
staff, 3, 3, file_secret, 4, 2, read         # staff（机密:3，完整:3）读取 file_secret（机密:4，完整:2）- 拒绝（conf < obj.conf）
manager, 4, 4, file_sensitive, 3, 5, read    # manager（机密:4，完整:4）读取 file_sensitive（机密:3，完整:5）- 拒绝（int < obj.int）
guest, 2, 2, file_internal, 3, 1, read       # guest（机密:2，完整:2）读取 file_internal（机密:3，完整:1）- 拒绝（conf < obj.conf）
staff, 3, 3, file_protected, 1, 4, read      # staff（机密:3，完整:3）读取 file_protected（机密:1，完整:4）- 拒绝（int < obj.int）

# 正常写入操作（允许）
guest, 2, 2, file_public, 2, 2, write        # guest（机密:2，完整:2）写入 file_public（机密:2，完整:2）- 允许
staff, 3, 3, file_internal, 5, 4, write      # staff（机密:3，完整:3）写入 file_internal（机密:5，完整:4）- 允许
manager, 4, 4, file_secret, 4, 5, write      # manager（机密:4，完整:4）写入 file_secret（机密:4，完整:5）- 允许
admin, 5, 5, file_archive, 5, 5, write       # admin（机密:5，完整:5）写入 file_archive（机密:5，完整:5）- 允许

# 写入操作违规（拒绝）
manager, 4, 4, file_internal, 3, 5, write    # manager（机密:4，完整:4）写入 file_internal（机密:3，完整:5）- 拒绝（conf > obj.conf）
staff, 3, 3, file_public, 2, 2, write        # staff（机密:3，完整:3）写入 file_public（机密:2，完整:2）- 拒绝（两者均高于对象）
admin, 5, 5, file_secret, 5, 4, write        # admin（机密:5，完整:5）写入 file_secret（机密:5，完整:4）- 拒绝（int > obj.int）
guest, 2, 2, file_private, 1, 3, write       # guest（机密:2，完整:2）写入 file_private（机密:1，完整:3）- 拒绝（conf > obj.conf）
```

## 安全级别

在此实现中，机密性和完整性均表示为整数，数值越高表示安全级别越高：

### 机密性级别

- 级别 1：公开/非机密
- 级别 2：机密
- 级别 3：秘密
- 级别 4：绝密

### 完整性级别

- 级别 1：低完整性（例如，公开数据、用户生成的内容）
- 级别 2：中等完整性（例如，已验证的数据、可信来源）
- 级别 3：高完整性（例如，系统数据、管理内容）
- 级别 4：关键完整性（例如，安全策略、系统配置）

## 适用场景

此 LBAC 实现特别适用于：

- 多级安全环境
- 数据保护和数据准确性都很重要的应用

## 实现说明

- 该模型实现了具有双重安全属性的强制访问控制（MAC）
- 系统管理员分配安全级别
- 访问决策同时考虑机密性和完整性级别
- 该模型防止信息泄露和数据损坏
