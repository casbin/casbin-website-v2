**管理API**涵盖了所有策略（Policy）和分组（角色）操作：获取、添加、删除、更新、过滤。下面，`e` 是一个执行器（Enforcer）实例。

## 过滤（Filtered）API

过滤方法接受 `(fieldIndex int, fieldValues ...string)` 参数：匹配从 `fieldIndex` 开始，`fieldValues` 是连续位置上所需的值。在 `fieldValues` 中使用 `""` 作为该位置的通配符。

Example:

```csv
p, alice, book, read
p, bob, book, read
p, bob, book, write
p, alice, pen, get
p, bob, pen ,get
```

```go
e.GetFilteredPolicy(1, "book") // returns: [["alice", "book", "read"], ["bob", "book", "read"], ["bob", "book", "write"]]

e.GetFilteredPolicy(1, "book", "read") // returns: [["alice", "book", "read"], ["bob", "book", "read"]]

e.GetFilteredPolicy(0, "alice", "", "read") // returns: [[alice book read]]

e.GetFilteredPolicy(0, "alice") // returns: [["alice", "book", "read"], ["alice", "pen", "get"]]
```

### `Enforce()`

Enforce 决定一个"主体（Subject）"是否可以通过操作"操作（Action）"访问一个"对象（Object）"，输入参数通常为：(sub, obj, act)。
