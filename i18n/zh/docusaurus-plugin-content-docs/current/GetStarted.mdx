## 安装

```mdx-code-block
<Tabs groupId="langs">
<TabItem value="Go" label="Go" default>
```

```bash
go get github.com/casbin/casbin/v3
```

:::info 从 v2 迁移到 v3

如果你正在从 Casbin v2 升级到 v3，请将导入路径从 `github.com/casbin/casbin/v2` 更新为 `github.com/casbin/casbin/v3`。这也适用于所有子包（例如，`v2/model` 变为 `v3/model`）。

对于 Go 模块，运行：

```bash
go get -u github.com/casbin/casbin/v3
```

然后将代码中所有 `/v2` 的导入更新为 `/v3`。
:::

### GraalVM 原生镜像支持

如果你正在使用 GraalVM 构建原生应用（例如使用 Quarkus 或 Spring Native），由于 jCasbin 使用了 [Aviator 表达式引擎](https://github.com/killme2008/aviatorscript)，需要进行特殊配置。

默认情况下，Aviator 通过 ASM 使用动态类生成，这在 GraalVM 原生镜像中不受支持。为了解决此问题，你必须将 Aviator 配置为使用解释器模式而非编译模式。

#### 针对 Quarkus 应用

将以下内容添加到你的 `application.properties` 或 `pom.xml` 中：

```xml
<properties>
    <quarkus.native.additional-build-args>
        -J-Daviator.eval.mode=INTERPRETER
    </quarkus.native.additional-build-args>
</properties>
```

#### 针对其他 GraalVM 原生构建

在构建原生镜像时设置系统属性：

```bash
-Daviator.eval.mode=INTERPRETER
```

或者在初始化 jCasbin 之前通过编程方式配置：

```java
System.setProperty("aviator.eval.mode", "INTERPRETER");
```

:::note
此配置将 Aviator 从默认的编译模式切换到解释器模式。虽然这可能会对性能产生轻微影响，但它通过避免运行时类生成，实现了与 GraalVM 原生镜像的完全兼容。
:::

## 创建 Casbin 执行器（Enforcer）

Casbin 依赖配置文件来指定访问控制（Access Control）模型（Model）。

你需要两个配置文件：`model.conf` 和 `policy.csv`。**模型（Model）**文件定义了你的访问控制模型；**策略（Policy）**文件中保存了权限（Permission）规则。在实际使用中，你只需要与一个组件打交道——**执行器（Enforcer）**——它在创建时加载这两个文件。

要创建执行器，需要提供一个[模型](/docs/supported-models)和一个[适配器（Adapter）](/docs/adapters)。

Casbin 提供了一个[FileAdapter](adapters#file-adapter-built-in)，你可以使用它。更多信息请参见[适配器](adapters)。

- 使用模型文本配合其他适配器的示例：

```mdx-code-block
<Tabs groupId="langs">
<TabItem value="Go" label="Go" default>
```

```go
import (
    "log"

    "github.com/casbin/casbin/v3"
    "github.com/casbin/casbin/v3/model"
    xormadapter "github.com/casbin/xorm-adapter/v2"
    _ "github.com/go-sql-driver/mysql"
)

// 使用 MySQL 数据库初始化 Xorm 适配器。
a, err := xormadapter.NewAdapter("mysql", "mysql_username:mysql_password@tcp(127.0.0.1:3306)/")
if err != nil {
    log.Fatalf("error: adapter: %s", err)
}

m, err := model.NewModelFromString(`
[request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = r.sub == p.sub && r.obj == p.obj && r.act == p.act
`)
if err != nil {
    log.Fatalf("error: model: %s", err)
}

e, err := casbin.NewEnforcer(m, a)
if err != nil {
    log.Fatalf("error: enforcer: %s", err)
}
```

### 检查权限

在访问资源之前，在代码中添加权限检查：

```mdx-code-block
<Tabs groupId="langs">
<TabItem value="Go" label="Go" default>
```

```go
sub := "alice" // 想要访问资源的用户
obj := "data1" // 将要被访问的资源
act := "read" // 用户对资源执行的操作

ok, err := e.Enforce(sub, obj, act)

if err != nil {
    // 处理错误
}

if ok == true {
    // 允许 alice 读取 data1
} else {
    // 拒绝请求，显示错误
}

// 你也可以使用 BatchEnforce() 批量执行请求。
// 该方法返回布尔值切片，切片索引与二维数组的行索引对应。
// 例如 results[0] 是 {"alice", "data1", "read"} 的执行结果
results, err := e.BatchEnforce([][]interface{}{{"alice", "data1", "read"}, {"bob", "data2", "write"}, {"jack", "data3", "read"}})
```
