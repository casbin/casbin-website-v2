Casbin 的插件（Plugin）生态系统通过存储后端、框架集成和分布式同步扩展了核心功能。使用插件将 Casbin 连接到您的数据库、消息总线或 Web 框架。

## 插件分类

| 类别（Category） | 用途 |
|----------------|---------|
| **执行器（Enforcer）**  | 针对特定环境或用例的专用执行器 |
| **适配器（Adapter）**   | 连接存储（数据库、文件、云端）以管理模型（Model）和策略（Policy） |
| **监视器（Watcher）**   | 在多个执行器实例间同步策略变更 |
| **调度器（Dispatcher）**| 协调分布式部署中的更新 |
| **角色管理器（Role Manager）** | 自定义角色/层次结构逻辑 |
| **中间件（Middleware）**| Kong、GraphQL 和云原生技术栈的集成 |

## 为什么使用插件？

- **集成** 您现有的数据库和基础设施
- **扩展** 使用监视器和调度器扩展到多个实例
- **使用** 框架特定的中间件（如 Nest.js、Gin）
- **运行** 在云原生和无服务器环境中

## 快速开始

1. 选择适合您需求的插件类型（适配器、监视器、中间件等）。
2. 安装适用于您所用语言的包。
3. 在创建或配置执行器时进行配置。
4. 查阅插件的文档以了解选项和高级用法。

使用本类别中的各章节来查找和配置每种插件类型。
