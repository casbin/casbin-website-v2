**RBAC API** 是一个用于基于角色操作的便捷层（用户的角色、角色的用户、权限等）。它是[管理API](/docs/management-api)的子集，专为RBAC使用场景而设计。

## 参考

### `GetNamedPermissionsForUser()`

GetNamedPermissionsForUser 通过命名策略获取用户或角色的权限。

For example:

```csv
p, alice, data1, read
p, bob, data2, write
p2, admin, create
```

### `GetAllowedObjectConditions()`

返回用户可以访问的对象条件。

For example:

```csv
p, alice, r.obj.price < 25, read
p, admin, r.obj.category_id = 2, read
p, bob, r.obj.author = bob, write

g, alice, admin
```

e.GetAllowedObjectConditions("alice", "read", "r.obj.") 将返回 `["price < 25", "category_id = 2"], nil`

注意：

0. 前缀：你可以自定义对象条件的前缀，"r.obj." 是常用的前缀。
移除前缀后，剩余部分即为对象条件。
如果存在不满足前缀要求的 obj 策略，将返回 `errors.ERR_OBJ_CONDITION`。

1. 如果 'objectConditions' 数组为空，返回 `errors.ERR_EMPTY_CONDITION`
返回此错误是因为某些数据适配器（Adapter）的 ORM 在收到空条件时默认返回全表数据，
这往往与预期行为相反（例如 GORM）。
如果你使用的适配器没有这种表现，可以选择忽略此错误。

