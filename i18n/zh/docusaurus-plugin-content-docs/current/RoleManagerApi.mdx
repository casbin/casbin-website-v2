**角色管理器**接口定义了角色层级和用户-角色链接的存储和查询方式。你可以附加**匹配函数**，从而支持角色名称或域名中的通配符（例如 `*`）。下面，`e` 是一个执行器（Enforcer），`rm` 是一个角色管理器。

## 角色管理器API

### AddNamedMatchingFunc

为策略类型（例如 `g`）注册一个匹配函数。在评估角色链接时使用，以便 `*` 等模式匹配多个名称。

```mdx-code-block
<Tabs groupId="langs">
<TabItem value="Go" label="Go" default>
```

```go
    e.AddNamedMatchingFunc("g", "", util.KeyMatch)
    _, _ = e.AddGroupingPolicies([][]string{{"*", "admin", "domain1"}})
    _, _ = e.GetRoleManager().HasLink("bob", "admin", "domain1") // -> true, nil
```

```mdx-code-block
</TabItem>
<TabItem value="Node.js" label="Node.js">
```

```typescript
    await e.addNamedMatchingFunc('g', Util.keyMatchFunc);
    await e.addGroupingPolicies([['*', 'admin', 'domain1']]);
    await e.getRoleManager().hasLink('bob', 'admin', 'domain1');
```

```mdx-code-block
</TabItem>
</Tabs>
```

### AddNamedDomainMatchingFunc

为角色链接中的**域**字段注册一个匹配函数（例如三个参数的 `g`）。支持在域名中使用通配符。

Example:

```mdx-code-block
<Tabs groupId="langs">
<TabItem value="Go" label="Go" default>
```

```go
    e, _ := casbin.NewEnforcer("path/to/model", "path/to/policy")
    e.AddNamedDomainMatchingFunc("g", "", util.MatchKey)
```

```mdx-code-block
</TabItem>
<TabItem value="Node.js" label="Node.js">
```

```typescript
    const e = await newEnforcer('path/to/model', 'path/to/policy');
    await e.addNamedDomainMatchingFunc('g', Util.keyMatchFunc);
```

```mdx-code-block
</TabItem>
</Tabs>
```
