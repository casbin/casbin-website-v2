本页面概述了Casbin的主要API。有关设置和概念，请参阅[入门](/docs/get-started)和[工作原理](/docs/how-it-works)。

## 执行（Enforce）API

以下示例使用了`model.conf`中的RBAC模型（Model）和`policy.csv`中的策略（Policy）。模型语法在[模型语法](/docs/syntax-for-models)中有详细说明。示例文件：

`model.conf`

```ini
[request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[role_definition]
g = _, _

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = g(r.sub, p.sub) && r.obj == p.obj && r.act == p.act
```

`policy.csv`

```csv
p, admin, data1, read
p, admin, data1, write
p, admin, data2, read
p, admin, data2, write
p, alice, data1, read
p, bob, data2, write
g, amber, admin
g, abc, admin
```

Example usage:

```go
// 从文件加载信息。
enforcer, err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    log.Fatalf("Error, detail: %s", err)
}
ok, err := enforcer.Enforce("alice", "data1", "read")
```

`NewEnforcer()` 从给定路径（或从模型和适配器）加载模型和策略。它返回一个执行器（Enforcer）和一个错误。当传入两个文件路径时，使用默认的文件适配器（Adapter）；你也可以替换为任何[适配器](/docs/adapters)。

`enforcer.Enforce("alice", "data1", "read")` 检查Alice是否可以读取data1。当允许时返回 `(true, nil)`，拒绝时返回 `(false, nil)`。使用上述策略，此调用返回 `true`。

## EnforceEx API

要查看哪条策略允许了请求，请使用 `EnforceEx()`：

```go
ok, reason, err := enforcer.EnforceEx("amber", "data1", "read")
fmt.Println(ok, reason) // true [admin data1 read]
```

`reason` 是匹配的策略（例如 `[admin data1 read]`）。Amber拥有 `admin` 角色（Role），因此策略 `p, admin, data1, read` 允许了该请求。

相关API：

- `ok, err := enforcer.EnforceWithMatcher(matcher, request)`

  使用一次性匹配器（Matcher）表达式执行。

- `EnforceExWithMatcher(matcher, request)` — 类似 `EnforceEx`，但使用自定义匹配器。

- `BatchEnforce(requests)` — 同时执行多个请求；返回布尔值切片。

详见[管理API](/docs/management-api)和[RBAC API](/docs/rbac-api)以获取完整列表。

## 管理（Management）API

### 获取（Get）API

这些方法返回策略数据。示例：

```go
enforcer, err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    fmt.Printf("Error, details: %s\n", err)
}
allSubjects := enforcer.GetAllSubjects()
fmt.Println(allSubjects)
```

`GetAllSubjects()` 返回策略中出现的所有主体（Subject）（此处为：`admin`、`alice`、`bob`）。输出：

```text
[admin alice bob]
```

对于特定的策略类型，使用 `GetAllNamedSubjects("p")`。同样的模式适用于**对象（Object）**、**操作（Action）**和**角色（Role）**。

其他获取方法：

- `GetPolicy()` / `GetFilteredPolicy(0, "alice")` — 所有规则或按字段过滤。
- `GetNamedPolicy("p")` / `GetFilteredNamedPolicy("p", 0, "bob")` — 同上，针对命名策略。
- `GetGroupingPolicy()` / `GetFilteredGroupingPolicy(0, "alice")` — 角色分配规则。
- `GetNamedGroupingPolicy("g")` / `GetFilteredNamedGroupingPolicy("g", 0, "alice")` — 同上，针对命名分组。

### 添加、删除、更新API

你可以在运行时更改策略。例如：添加、删除、更新和检查：

```go
// 从文件加载信息
enforcer, err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    fmt.Printf("Error, details: %s\n", err)
}

// 添加策略并使用 HasPolicy() 确认
enforcer.AddPolicy("added_user", "data1", "read")
hasPolicy := enforcer.HasPolicy("added_user", "data1", "read")
fmt.Println(hasPolicy) // true，策略已成功添加

// 移除策略并使用 HasPolicy() 确认
enforcer.RemovePolicy("alice", "data1", "read")
hasPolicy = enforcer.HasPolicy("alice", "data1", "read")
fmt.Println(hasPolicy) // false，策略已成功移除

// 更新策略并使用 HasPolicy() 确认
enforcer.UpdatePolicy([]string{"added_user", "data1", "read"}, []string{"added_user", "data1", "write"})
hasPolicy = enforcer.HasPolicy("added_user", "data1", "read")
fmt.Println(hasPolicy) // false，原策略已过期
hasPolicy = enforcer.HasPolicy("added_user", "data1", "write")
fmt.Println(hasPolicy) // true，新策略已生效
```

同样的添加/删除/更新/检查模式也适用于过滤策略、命名策略和分组策略。对于批量更新，请使用复数形式（例如 `AddPolicies`、`UpdatePolicies`），参数为切片。

Example:

```go
enforcer.UpdatePolicy([]string{"eve", "data3", "read"}, []string{"eve", "data3", "write"})
```

将 `Policy` 改为 `Policies` 并修改参数即可进行批量操作：

```go
enforcer.UpdatePolicies([][]string{{"eve", "data3", "read"}, {"jack", "data3", "read"}}, [][]string{{"eve", "data3", "write"}, {"jack", "data3", "write"}})
```

批量操作也适用于 `GroupingPolicy` 和 `NamedGroupingPolicy`。

### AddEx API

AddEx 方法批量添加规则，但会**跳过**已存在的规则而不是失败：

```go
AddPoliciesEx(rules [][]string) (bool, error)
AddNamedPoliciesEx(ptype string, rules [][]string) (bool, error)
AddGroupingPoliciesEx(rules [][]string) (bool, error)
AddNamedGroupingPoliciesEx(ptype string, rules [][]string) (bool, error)
SelfAddPoliciesEx(sec string, ptype string, rules [][]string) (bool, error) 
```

使用 `AddPolicies` 时，如果某条规则已存在，整个调用将失败且不会添加任何规则。而使用 `AddPoliciesEx` 时，会跳过已存在的规则并添加其余规则。示例：

```go
func TestDemo(t *testing.T) {
    e, err := NewEnforcer("examples/basic_model.conf", "examples/basic_policy.csv")
    if err != nil {
        fmt.Printf("Error, details: %s\n", err)
    }
    e.ClearPolicy()
    e.AddPolicy("user1", "data1", "read")
    fmt.Println(e.GetPolicy())
    testGetPolicy(t, e, [][]string{{"user1", "data1", "read"}})

    // 策略 {"user1", "data1", "read"} 现已存在

    // 使用 AddPolicies 批量添加规则
    ok, _ := e.AddPolicies([][]string{{"user1", "data1", "read"}, {"user2", "data2", "read"}})
    fmt.Println(e.GetPolicy())
    // {"user2", "data2", "read"} 添加失败，因为 {"user1", "data1", "read"} 已存在
    // AddPolicies 返回 false，不会检查其他策略，即使它们可能不在现有规则集中
    // ok == false
    fmt.Println(ok)
    testGetPolicy(t, e, [][]string{{"user1", "data1", "read"}})

    // 使用 AddPoliciesEx 批量添加规则
    ok, _ = e.AddPoliciesEx([][]string{{"user1", "data1", "read"}, {"user2", "data2", "read"}})
    fmt.Println(e.GetPolicy())
    // {"user2", "data2", "read"} 成功添加
    // 因为 AddPoliciesEx 自动过滤了已存在的 {"user1", "data1", "read"}
    // ok == true
    fmt.Println(ok)
    testGetPolicy(t, e, [][]string{{"user1", "data1", "read"}, {"user2", "data2", "read"}})
}
```

## RBAC API

执行器还提供了RBAC辅助方法（角色、用户、权限）。有关模型设置，请参阅 [RBAC](/docs/rbac)。

Load model and policy as before:

```go
enforcer, err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    fmt.Printf("Error, details: %s\n", err)
}
```

Example RBAC calls:

```go
roles, err := enforcer.GetRolesForUser("amber")
fmt.Println(roles) // [admin]
users, err := enforcer.GetUsersForRole("admin")
fmt.Println(users) // [amber abc]
```

`GetRolesForUser("amber")` 返回 `[admin]`。`GetUsersForRole("admin")` 返回拥有该角色的所有用户。`HasRoleForUser("amber", "admin")` 返回 `true`。

```go
fmt.Println(enforcer.Enforce("bob", "data2", "write")) // true
enforcer.DeletePermission("data2", "write")
fmt.Println(enforcer.Enforce("bob", "data2", "write")) // false
```

`DeletePermission("data2", "write")` 会删除所有人的该权限。`DeletePermissionForUser("alice", "data1", "read")` 仅删除Alice的该权限。

有关RBAC API的完整列表，请参阅 [RBAC API](/docs/rbac-api)。
