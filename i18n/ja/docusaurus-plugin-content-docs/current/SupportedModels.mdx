1. [**ACL (アクセス制御リスト)**](https://en.wikipedia.org/wiki/Access_control_list)
2. **[スーパーユーザー](https://en.wikipedia.org/wiki/Superuser) を含む ACL**
3. **ユーザーがいないACL**: 認証やユーザーログインがないシステムでは特に便利です。
3. **ACL without resources**: some scenarios may target for a type of resources instead of an individual resource by using permissions like `write-article`, `read-log`. 特定の記事やログへのアクセスを制御しません。
4. **[RBAC (ロールベースのアクセス制御)](https://en.wikipedia.org/wiki/Role-based_access_control)**
5. **リソース ロールを持つ RBAC**: ユーザとリソースの両方が同時にロール (またはグループ) を持つことができます。
6. **ドメイン/テナントを持つ RBAC**: ユーザーは、異なるドメイン/テナントに異なるロールセットを持つことができます。
7. **[ABAC (Attribute-Based Access Control)](https://en.wikipedia.org/wiki/Attribute-Based_Access_Control)**: resource.Owner `のような構文シュガー` を使用してリソースの属性を取得できます。
8. **[RESTful](https://en.wikipedia.org/wiki/Representational_state_transfer)**: supports paths like `/res/*`, `/res/:id` and HTTP methods like `GET`, `POST`, `PUT`, `DELETE`.
9. **拒否オーバーライド**: 許可と拒否の両方がサポートされており、許可の上書きを拒否します。
10. **優先度**: ポリシールールはファイアウォールルールのように優先順位付けできます。
