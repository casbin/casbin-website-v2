1. [**ACL (Zugriffskontrollliste)**](https://en.wikipedia.org/wiki/Access_control_list)
2. **ACL mit [Superuser](https://en.wikipedia.org/wiki/Superuser)**
3. **ACL ohne Benutzer**: besonders nützlich für Systeme, die keine Authentifizierung oder Benutzeranmeldung haben.
3. **ACL ohne Ressourcen**: Einige Szenarien können auf eine Art von Ressourcen anstatt auf eine einzelne Ressource zielen, indem sie Berechtigungen wie `Write-Artikel`verwenden, `Leseprotokoll`. Es kontrolliert nicht den Zugriff auf einen bestimmten Artikel oder Log.
4. **[RBAC (Rollenbasierte Zugriffssteuerung)](https://en.wikipedia.org/wiki/Role-based_access_control)**
5. **RBAC mit Ressourcenrollen**: Benutzer und Ressourcen können Rollen (oder Gruppen) gleichzeitig haben.
6. **RBAC mit Domänen/Mietern**: Benutzer können unterschiedliche Rollensätze für verschiedene Domains/Mieter haben.
7. **[ABAC (Attribute-Based Access Control)](https://en.wikipedia.org/wiki/Attribute-Based_Access_Control)**: Syntaxzucker wie `Ressource.Besitzer` kann verwendet werden, um das Attribut für eine Ressource zu erhalten.
8. **[RESTful](https://en.wikipedia.org/wiki/Representational_state_transfer)**: unterstützt Pfade wie `/res/*`, `/res/:id` und HTTP-Methoden wie `GET`, `POST`, `PUT`, `LÖSCHEN`.
9. **'Deny-override'**: Genehmigungen und Verweigern von Genehmigungen werden unterstützt, verweigert das Zulassungsrecht.
10. **Priorität**: Die Richtlinien-Regeln können wie Firewall-Regeln priorisiert werden.
